## UC-01. Регистрация и вход

**Актор:** новый или существующий пользователь.

**Предусловие:** пользователь открыл приложение в браузере (мобильный или десктоп).

**Основной сценарий:**

1. Пользователь нажимает «Войти» или «Создать аккаунт».
2. Выбирает способ: email + пароль или OAuth (Google / Яндекс).
3. При регистрации: вводит email, задаёт пароль. (MVP: без подтверждения email; подтверждение — в backlog.)
4. При входе: вводит email + пароль или нажимает кнопку OAuth-провайдера.
5. После успешной авторизации — попадает на экран «Мои проекты».

**Альтернативные сценарии:**

- Забыл пароль → «Восстановить пароль» → письмо с ссылкой на сброс (требует настройки SMTP, реализуется в Sprint 5).
- Ошибка ввода → сообщение об ошибке, повторная попытка.

## UC-02. Создание нового проекта

**Актор:** авторизованный пользователь.

**Предусловие:** пользователь находится на экране «Мои проекты».

**Основной сценарий:**

1. Пользователь нажимает «Новый проект».
2. Вводит название проекта (например, «Квартира на Ленина»).
3. Опционально: загружает фото планировки от застройщика (для справки).
4. Проект создаётся, пользователь переходит к визарду (UC-03).

**Результат:** проект сохранён на сервере, привязан к аккаунту.

## UC-03. Прохождение визарда обмера (главный сценарий)

**Актор:** авторизованный пользователь.

**Предусловие:** проект создан (UC-02).

Визард состоит из 5 шагов. Прогресс автоматически сохраняется на сервер. Пользователь может прервать работу и вернуться позже — всё сохранено.

### Шаг 1. Общая информация о квартире

1. Тип объекта (квартира, студия, апартаменты, дом — выпадающий список).
2. Адрес (текстовое поле, опционально).
3. Высота потолков «по умолчанию» (значение, которое подставится во все комнаты — можно изменить для каждой).

### Шаг 2. Добавление помещений

1. Пользователь добавляет комнаты по одной: выбирает тип (кухня, спальня, санузел, коридор, балкон, кладовая и т.д.) + задаёт понятное название.
2. Список комнат отображается карточками. Можно удалить или переименовать.
3. Подсказка: «Не забудьте про коридор, балкон и кладовую!».

### Шаг 3. Обмер помещения (повторяется для каждой комнаты)

Это самый объёмный шаг. Для каждой комнаты пользователь последовательно проходит подшаги:

**3.1. Разметка углов**
- Пользователь размечает углы комнаты: A (левый нижний), далее по часовой стрелке B, C, D и т.д.
- Для прямоугольной комнаты — 4 угла. Для Г-образной — 6. Для П-образной — 8.
- UI показывает шаблон формы комнаты (из шага 2), пользователь подтверждает или корректирует.
- Подсказка: «Встаньте у входа, определите левый нижний угол — это точка A. Далее по часовой стрелке.»

**3.2. Высота потолка**
- 2 замера: у окна + у противоположной стены.
- Если разница > 10мм — подсветить предупреждение: «Перепад потолка {X}мм — проверьте замер или зафиксируйте как есть».

**3.3. Габариты стен**
- Общие длины стен: A-B, B-C (для прямоугольных — 2 стены, остальные = противоположным).
- Для сложных форм — длины всех стен.
- Для каждой стены: материал (бетон, ГКЛ/ГВЛ, кирпич, другое), тип (внешняя / внутренняя / смежная).

**3.4. Обход периметра**
- От угла A по часовой стрелке. Пользователь последовательно добавляет сегменты вдоль каждой стены.
- Типы сегментов: стена (простенок), окно, дверь, выступ, ниша, перегородка.
- Для выступов и ниш — дополнительно указывается глубина.
- UI показывает текущую стену и схему с уже добавленными сегментами.
- Автовалидация: сумма длин сегментов сравнивается с длиной стены из шага 3.3. Если расхождение > 20мм — предложить перемерить.
- Подсказка с примером: пронумерованные сегменты на плане комнаты.

**3.5. Проёмы и откосы**
- Для каждого окна (найденного при обходе): ширина (уже из сегмента), высота, высота подоконника от стяжки, откосы (глубина лево/право). Фото.
- Для каждой двери: ширина (уже из сегмента), высота от стяжки, откосы. Фото.
- Пометка для французских/балконных дверей.

**3.6. Развёртка стен**
Последовательно для каждой стены: A-B → B-C → C-D → D-A.
UI показывает «полотно» стены (прямоугольник: длина × высота потолка) с уже размещёнными проёмами из шагов 3.4-3.5.

На каждой развёртке пользователь замеряет и вносит:
- Вертикальные привязки проёмов: верх окна до потолка, высота двери.
- Вертикальные размеры выступов/ниш: высота от пола, высота элемента.
- Элементы на стене: радиатор, электрощиток, слаботочный щиток, вент-шахта, стояк — позиция (от начала стены, от пола), размеры (ширина × высота × глубина), отступ от стены. Для угловых элементов (вент-шахта в углу) — пометка «угловой, привязан к углу X».
- Кривизна стены: 3 замера (низ, середина, верх) — приложение считает среднее и отклонение.
- Фото стены (опционально).

**3.7. Фото + чеклист**
- Обязательное фото общего вида комнаты после всех замеров.
- Чек-лист: все стены замерены? Все проёмы? Все элементы? Кривизна проверена?
- Кнопка «Готово с комнатой».

### Шаг 4. Сборка плана квартиры

1. Приложение показывает список всех обмеренных комнат.
2. Пользователь указывает связи: комната А → стена N → примыкает к комнате Б → стена M. Отмечает дверные проёмы между комнатами.
3. Приложение автоматически строит 2D-план (SVG) на основе связей и размеров стен.
4. Пользователь проверяет план визуально. Доступно: перетаскивание комнат, поворот на 90°, ручная подвижка стены.
5. Кнопка «Всё правильно» или «Исправить связь».

### Шаг 5. Проверка и финальный обмерный план

1. Приложение показывает сводку: список всех комнат с ключевыми размерами, общие площади, периметры, объёмы.
2. Пользователь может вернуться к любой комнате и исправить данные.
3. Кнопка «Сформировать обмерный план».

## UC-04. Просмотр обмерного плана

**Актор:** авторизованный пользователь.

**Предусловие:** визард завершён (UC-03, шаг 5).

**Основной сценарий:**

1. Открывается интерактивный веб-дашборд:
   - 2D-схема квартиры. Клик на комнату → все размеры и фото этой комнаты.
   - Развёртки стен с элементами и кривизной.
   - Таблицы по каждому помещению (стены, окна, двери, элементы).
   - Общие итоги: площадь квартиры, площадь каждой комнаты, периметры, объёмы.
2. Кнопка «Скачать PDF» — генерация обмерного плана в PDF (с водяным знаком в MVP).

## UC-05. Управление проектами

**Актор:** авторизованный пользователь.

**Предусловие:** пользователь на экране «Мои проекты».

**Сценарии:**

- Просмотр списка всех проектов (название, дата создания, статус: черновик / завершён).
- Открытие проекта — продолжение работы с того места, где остановился.
- Удаление проекта (с подтверждением).
- Дублирование проекта (для повторного обмера или шаблона).

---

## UC-06. Редактирование завершённого проекта

**Актор:** авторизованный пользователь.

**Предусловие:** проект завершён (план сформирован).

**Основной сценарий:**

1. Пользователь открывает завершённый проект.
2. Нажимает «Редактировать».
3. Может изменить замеры любой комнаты, добавить/удалить комнату.
4. После изменений — пересборка плана (шаг 4–5 визарда).
