## B-01 — Затемнённый экран после закрытия модалки

**Дата:** 2026-04-09  
**Статус:** ✓ Закрыт  
**Коммит:** `420307e`

### Симптом

После авторизации и закрытия модального окна (CreateProjectModal) вся страница оставалась затемнённой (серый оверлей), интерфейс не реагировал на клики.

### Root cause

`<dialog>` в `Modal.tsx` рендерился условно: `if (!isOpen) return null`. При закрытии модалки React убирал компонент из DOM **до** того, как `useEffect` успевал вызвать `dialog.close()`. В результате `::backdrop` pseudo-элемент оставался в browser top layer как orphan — без родительского `<dialog>`.

### Затронутый файл

`client/src/components/ui/Modal.tsx`

### Фикс

- Убран ранний `return null` — `<dialog>` всегда присутствует в DOM
- Контент рендерится условно внутри `{isOpen && (<>...</>)}`
- `showModal()` / `close()` используют optional chaining для совместимости с jsdom в тестах

### Тесты

289 тестов, все прошли (`npm test --run`)

## B-02 — Кнопка "Далее" заблокирована; запятая не принимается в поле высоты

**Дата:** 2026-04-09  
**Статус:** ✓ Закрыт  
**Коммит:** `f351c93`

### Симптом

На шаге "Общая информация": кнопка "Далее →" неактивна даже при корректно заполненной форме. Поле "Высота потолков" не принимает значения с запятой (`2,99`).

### Root cause

1. `disabled={isSubmitting || !isDirty}` — блокировка при отсутствии изменений. После загрузки существующего проекта форма заполнена, но `isDirty = false` → кнопка всегда неактивна.
2. `type="number"` + `z.coerce.number()` принимают только точку как десятичный разделитель. Русская локаль вводит запятую.

### Затронутый файл

`client/src/pages/wizard/ProjectInfoStep.tsx`

### Фикс

- Убран `!isDirty` из `disabled`; при `!isDirty` — navigate без API-вызова
- Поле изменено на `type="text" inputMode="decimal"` + Zod preprocess: `val.replace(',', '.')`

### Тесты

289 тестов, все прошли

## B-03 — Неверная нумерация углов; метки перекрывают кружки

**Дата:** 2026-04-09  
**Статус:** ✓ Закрыт  
**Коммит:** `e014cd0`

### Симптом

На шаге 3.1 «Разметка углов»:
1. Угол A не совпадал с нижним левым (неверный порядок для некоторых форм).
2. Буквенные метки A–H накладывались на кружки углов, делая их нечитаемыми.
3. Подсказка отправляла пользователя к «дальнему» углу, хотя это не нужно.

### Root cause

- SVG-координаты текстовых меток совпадали с координатами `<circle>`, текст рисовался поверх точки.
- Порядок обхода углов не соответствовал правилу «A = нижний левый, по часовой стрелке».
- Для L_SHAPE и U_SHAPE отсутствовали элементы `<circle>`.

### Затронутый файл

`client/src/pages/wizard/measure/CornerLabelStep.tsx`

### Фикс

- Пересчитаны координаты всех меток: смещены за границу соответствующего кружка.
- Все четыре формы приведены к единому правилу A = BL → по часовой.
- Добавлены `<circle>` для L_SHAPE и U_SHAPE.
- Подсказка обновлена: «угол A — ближайший слева у входа, поворот налево».

### Тесты

289 (client) + 208 (server) тестов, все прошли

## B-04 — RoomCard: имя и тип комнаты обрезаются некорректно

**Дата:** 2026-04-10  
**Статус:** ✓ Закрыт  
**Коммит:** `81bca54`

### Симптом

В списке комнат (RoomsStep) карточка показывает обрезанное имя («К.») и статус «Не замерена» вместо полного имени и типа. Длинные названия типов не умещаются в строку.

### Root cause

В `RoomCard` контейнер `flex-1 min-w-0` обрезает имя через `truncate`, но нижняя строка (тип + статус) не имеет гибкого ограничения — при длинном типе бейдж статуса вытесняет текст типа.

### Затронутый файл

`client/src/pages/wizard/RoomsStep.tsx` — компонент `RoomCard`

## B-05 — Прогресс-бар под-шагов MeasureStep выходит за границы экрана на мобильном

**Дата:** 2026-04-10  
**Статус:** ✓ Закрыт  
**Коммит:** `81bca54`

### Симптом

На экране 375px прогресс-бар (7 кружков + 6 линий + метка) по ширине ≈ 400px — появляется горизонтальный скролл или элементы обрезаются.

### Root cause

В `MeasureStep.tsx` прогресс-бар рендерится как `flex items-center gap-1` в одну строку без `overflow-hidden` и без адаптации под мобильный экран. 7 × 28px + 6 × 24px + gap + label > 375px.

### Затронутый файл

`client/src/pages/wizard/MeasureStep.tsx` — блок прогресс-бара

## B-06 — Ошибка «height must be a positive number» в PerimeterWalkStep

**Дата:** 2026-04-10  
**Статус:** ✓ Закрыт  
**Коммит:** `ca23f7e`

### Симптом

При добавлении сегмента стены появляется ошибка «height must be a positive number», хотя поля height в видимой форме нет.

### Root cause

В `PerimeterWalkStep.tsx` Zod discriminated union содержит ветку с обязательным `height: z.number().positive()`. При смене `segmentType` форма могла сохранять поля предыдущей ветки; `z.discriminatedUnion` на submit проверяет `height` ветки, но поле не отображается, и значение = undefined.

### Затронутый файл

`client/src/pages/wizard/measure/PerimeterWalkStep.tsx`

## B-07 — OpeningsStep: поля высоты ожидают мм, но пользователь вводит метры

**Дата:** 2026-04-10  
**Статус:** ✓ Закрыт

### Симптом

Поля «Высота проёма» и «Высота подоконника» в шаге 3.5 принимают целые числа в мм, тогда как во всех остальных шагах пользователь вводит метры (с конвертацией на слое API). Пользователь вводит «0.9» или «0.015», получает ошибку «≥ 0» или неверные данные.

### Root cause

В `OpeningsStep.tsx` использованы `type="number" step="1" min="0"` и Zod-схема с `z.coerce.number().min(0)` без умножения на 1000. Для согласованности с `WallDimensionsStep` и `PerimeterWalkStep` нужно принимать метры и конвертировать в мм перед отправкой.

### Затронутый файл

`client/src/pages/wizard/measure/OpeningsStep.tsx`

## B-08 — Фотографии не отображаются в PhotoChecklistStep (broken image)

**Дата:** 2026-04-10  
**Статус:** ✓ Закрыт  
**Коммит:** `426f96e`

### Симптом

В шаге 3.7 «Фото» добавленные фотографии показывают иконку broken image вместо превью.

### Root cause

URL превью (`thumbPath` или `originalPath`) из API возвращается как относительный путь (`/uploads/...`). При рендере в `<img src={...}>` браузер резолвит его от `window.location.origin`, но dev-сервер Vite не проксирует `/uploads/` на бэкенд. В продакшне Nginx тоже может не отдавать статику по правильному пути.

### Затронутый файл

`client/src/pages/wizard/measure/PhotoChecklistStep.tsx`, `client/vite.config.ts` (proxy), `nginx/nginx.conf` (static serving)
