## B-01 — Затемнённый экран после закрытия модалки

**Дата:** 2026-04-09  
**Статус:** ✓ Закрыт  
**Коммит:** `420307e`

### Симптом

После авторизации и закрытия модального окна (CreateProjectModal) вся страница оставалась затемнённой (серый оверлей), интерфейс не реагировал на клики.

### Root cause

`<dialog>` в `Modal.tsx` рендерился условно: `if (!isOpen) return null`. При закрытии модалки React убирал компонент из DOM **до** того, как `useEffect` успевал вызвать `dialog.close()`. В результате `::backdrop` pseudo-элемент оставался в browser top layer как orphan — без родительского `<dialog>`.

### Затронутый файл

`client/src/components/ui/Modal.tsx`

### Фикс

- Убран ранний `return null` — `<dialog>` всегда присутствует в DOM
- Контент рендерится условно внутри `{isOpen && (<>...</>)}`
- `showModal()` / `close()` используют optional chaining для совместимости с jsdom в тестах

### Тесты

289 тестов, все прошли (`npm test --run`)

## B-02 — Кнопка "Далее" заблокирована; запятая не принимается в поле высоты

**Дата:** 2026-04-09  
**Статус:** ✓ Закрыт  
**Коммит:** `f351c93`

### Симптом

На шаге "Общая информация": кнопка "Далее →" неактивна даже при корректно заполненной форме. Поле "Высота потолков" не принимает значения с запятой (`2,99`).

### Root cause

1. `disabled={isSubmitting || !isDirty}` — блокировка при отсутствии изменений. После загрузки существующего проекта форма заполнена, но `isDirty = false` → кнопка всегда неактивна.
2. `type="number"` + `z.coerce.number()` принимают только точку как десятичный разделитель. Русская локаль вводит запятую.

### Затронутый файл

`client/src/pages/wizard/ProjectInfoStep.tsx`

### Фикс

- Убран `!isDirty` из `disabled`; при `!isDirty` — navigate без API-вызова
- Поле изменено на `type="text" inputMode="decimal"` + Zod preprocess: `val.replace(',', '.')`

### Тесты

289 тестов, все прошли

## B-03 — Неверная нумерация углов; метки перекрывают кружки

**Дата:** 2026-04-09  
**Статус:** ✓ Закрыт  
**Коммит:** `e014cd0`

### Симптом

На шаге 3.1 «Разметка углов»:
1. Угол A не совпадал с нижним левым (неверный порядок для некоторых форм).
2. Буквенные метки A–H накладывались на кружки углов, делая их нечитаемыми.
3. Подсказка отправляла пользователя к «дальнему» углу, хотя это не нужно.

### Root cause

- SVG-координаты текстовых меток совпадали с координатами `<circle>`, текст рисовался поверх точки.
- Порядок обхода углов не соответствовал правилу «A = нижний левый, по часовой стрелке».
- Для L_SHAPE и U_SHAPE отсутствовали элементы `<circle>`.

### Затронутый файл

`client/src/pages/wizard/measure/CornerLabelStep.tsx`

### Фикс

- Пересчитаны координаты всех меток: смещены за границу соответствующего кружка.
- Все четыре формы приведены к единому правилу A = BL → по часовой.
- Добавлены `<circle>` для L_SHAPE и U_SHAPE.
- Подсказка обновлена: «угол A — ближайший слева у входа, поворот налево».

### Тесты

289 (client) + 208 (server) тестов, все прошли
