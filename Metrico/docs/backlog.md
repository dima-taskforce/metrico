## F-01 — Классификатор адресов + типовые планы квартир

**Приоритет:** Средний  
**Статус:** Идея

### Описание

Два связанных улучшения на шаге "Общая информация":

1. **Автоподстановка адреса** — интеграция с API классификатора адресов (DaData, ФИАС, либо Google Places) для автодополнения поля "Адрес". Пользователь начинает вводить улицу — получает подсказки с нормализованным адресом.

2. **Типовые планы квартир** — после выбора адреса (или серии дома) автоматически предлагать готовый план-шаблон. Источники: открытые базы типовых серий (П-44, П-3, II-18 и др.), парсинг planDB / dom.mingkh.ru, либо собственная база шаблонов.

### Ценность

- Ускоряет создание проекта: меньше ручного ввода
- Типовой план даёт стартовую точку для обмера — пользователь корректирует, а не создаёт с нуля
- Снижает ошибки в адресах (нормализация)

### Возможная реализация

| Компонент | Вариант |
|-----------|---------|
| Автодополнение адреса | DaData API (suggestions/address) — платный, есть free tier |
| Альтернатива | Nominatim (OpenStreetMap) — бесплатный, менее точный для РФ |
| Типовые планы | Собственная база JSON-шаблонов по серии дома |
| Определение серии | По адресу → год постройки → серия (открытые данные Росреестра / ДомКлик) |

### Зависимости

- Требует отдельного API-ключа (DaData или аналог)
- Формат шаблона плана должен совпадать со схемой Prisma (комнаты, стены, проёмы)

## F-03 — Заменить эмоджи на иконки Phosphor

**Приоритет:** Средний  
**Статус:** Идея

### Описание

Заменить все эмоджи в UI на иконки из пакета `@phosphor-icons/react`. Эмоджи рендерятся по-разному на разных ОС и выглядят непрофессионально.

### Установка

```bash
npm install @phosphor-icons/react --workspace=client
```

### Затронутые места

Найти все эмоджи в клиентском коде:

```bash
rg "[^\x00-\x7F]" client/src --type tsx -l
```

Ожидаемые места:
- `RoomsStep.tsx` — эмоджи типов комнат (🛋️ 🛏️ 🚿 и др.)
- Любые другие компоненты с эмоджи-иконками

### Подход

- Каждому эмоджи подобрать семантически близкую иконку Phosphor
- Использовать вес `Regular` по умолчанию, `Bold` для акцентов
- Размер: `size={20}` для inline, `size={24}` для кнопок

## F-02 — Дополнительные типы комнат

**Приоритет:** Низкий  
**Статус:** Идея  
**Связать с:** ближайшей задачей, затрагивающей RoomType

### Описание

Добавить в enum `RoomType` два новых типа:

- `KITCHEN_LIVING` — Кухня-гостиная

- `LIBRARY` — Библиотека
