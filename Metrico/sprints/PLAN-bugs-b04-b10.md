### Шаг 2. B-05 — MeasureStep progress bar (MeasureStep.tsx)

**Задача:** Сделать прогресс-бар адаптивным под мобильный.

**Решение A (предпочтительное):** Уменьшить кружки с `w-7 h-7` до `w-6 h-6`, линии с `w-6` до `w-3`, метку перенести на следующую строку (`flex-wrap` или отдельный `<div>`).

**Решение B:** Заменить числа на точки (минимальный dot-indicator) и показывать label отдельно.

Итоговая ширина: 7×24px + 6×12px + gaps ≈ 240px — умещается в 375px.

**Тест:** тест на рендер заголовка MeasureStep, проверка отсутствия overflow

### Шаг 3. B-06 — PerimeterWalkStep height error (PerimeterWalkStep.tsx)

**Задача:** Найти источник ошибки «height must be a positive number».

**Гипотеза:** Ошибка приходит с сервера (`400 Bad Request`) при отправке сегмента с `height: undefined/null`. Бэкенд DTO имеет поле `height` как required.

**Действия:**
1. Проверить DTO `CreateWallSegmentDto` в `server/src/walls/`
2. Если `height` обязателен на бэкенде — сделать опциональным или удалить из DTO
3. Если проблема во фронтовой Zod-схеме — проверить discriminated union

**Тест:** E2E тест добавления сегмента PLAIN без height

### Шаг 5. B-08 — Broken image (PhotoChecklistStep + vite.config + nginx)

**Задача:** Исправить отображение загруженных фото.

**Dev (Vite proxy):**
```ts
// vite.config.ts
proxy: {
  '/api': { target: 'http://localhost:3000', ... },
  '/uploads': { target: 'http://localhost:3000', ... },  // добавить
}
```

**Prod (Nginx):**
```nginx
location /uploads/ {
  alias /app/uploads/;  # путь к uploads на сервере
}
```

**Проверить** в `PhotoChecklistStep.tsx`: `<img src={photo.thumbPath || photo.originalPath}>` — убедиться, что путь корректный.

**Тест:** smoke-тест загрузки и отображения фото
