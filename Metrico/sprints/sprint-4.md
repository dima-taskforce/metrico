## S4-01. Интерактивный дашборд (UC-04)

**Агент:** react-expert

**Описание:**
Реализовать страницу `/projects/:id/plan` — интерактивный просмотр обмерного плана:
- 2D-схема квартиры (React-Konva, из svgData/layoutJson).
- Клик по комнате → боковая панель с деталями: все размеры, фото, элементы.
- Таблицы по каждому помещению (стены, окна, двери, элементы).
- Общие итоги: площадь квартиры, площадь каждой комнаты, периметры, объёмы.
- Легенда с условными обозначениями (как на профессиональном обмерном плане).
- Кнопка «Скачать PDF».
- Кнопка «Редактировать» → возврат в визард (UC-06).

Mobile-first: на телефоне — план сверху (с жестами), детали снизу (свайп вверх).

**Тесты:**
- Рендеринг плана. Клик по комнате → детали. Таблицы. Итоги.

**Зависимости:** Sprint 3 завершён.

## S4-02. Генерация PDF (PdfGeneratorService)

**Агент:** fullstack

**Описание:**
Реализовать `PdfGeneratorService` на бэкенде:
- `GET /api/projects/:id/plan/pdf` — генерирует PDF и отдаёт файл.
- Использовать `@react-pdf/renderer` (SSR на сервере).

**Инфраструктурная настройка (обязательный первый шаг):**
1. Создать `server/tsconfig.pdf.json` с `"jsx": "react-jsx"` (или расширить основной tsconfig).
2. Установить `@react-pdf/renderer@^3` и `react` (peer dependency) в server workspace.
3. Проверить совместимость версий React между client и server (версии должны совпадать или быть совместимыми).
4. Создать тестовый PDF (пустой документ) и убедиться, что `renderToBuffer()` работает в NestJS-контексте.
5. Если возникают конфликты — рассмотреть альтернативу: `puppeteer` (headless Chrome) для генерации PDF из HTML-шаблона.

Содержание PDF (по образцу профессионального обмерного плана):
- Титульная страница: «Обмерный план», адрес, дата.
- Страница с 2D-схемой квартиры (SVG → растеризация или вставка).
- Условные обозначения (легенда): радиатор, электрощит, вент-шахта, трубы, высоты.
- По каждой комнате — отдельная страница:
  - Название, тип, площадь, периметр, объём.
  - Таблица стен (длина, материал, кривизна).
  - Таблица окон и дверей (размеры, простенки).
  - Таблица элементов.
  - Фото (превью).
- Итоговая страница: общая площадь, сводная таблица всех помещений.
- Водяной знак «Metrico MVP» на каждой странице.

**Тесты:**
- Генерация PDF для seed-квартиры: файл создаётся, размер > 0, количество страниц корректно.
- Watermark присутствует.

**Зависимости:** S4-01, S3-02 (расчёты), S2-05 (фото).

## S4-05. Управление проектами (доработка)

**Агент:** fullstack

**Описание:**
Доработать страницу «Мои проекты» и бэкенд:
- Статус проекта: DRAFT → COMPLETED. **Триггер:** модифицировать `PlanAssemblerService.assemble()` (из S3-03) — добавить вызов `projectsService.updateStatus(id, 'COMPLETED')` после успешной сборки. Без успешной сборки плана статус остаётся DRAFT. Генерация PDF возможна только для проектов со статусом COMPLETED.
- Дублирование проекта (UC-05): кнопка → глубокая копия. **Важно:** при дублировании `FileStorageService.copyProjectFiles(srcProjectId, dstProjectId)` физически копирует все фото из `uploads/{userId}/{srcProjectId}/` в `uploads/{userId}/{dstProjectId}/`, обновляя пути в скопированных записях БД.
- Редактирование завершённого проекта (UC-06): «Редактировать» → статус обратно в DRAFT, переход в визард.
- `GET /api/projects/:id` — возвращает полные данные для восстановления состояния визарда.

**Тесты:**
- Смена статуса. Дублирование: проверка, что все данные скопированы. Редактирование завершённого проекта.

**Зависимости:** S3-03, S4-02.
