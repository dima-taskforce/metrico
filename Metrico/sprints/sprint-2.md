# Sprint 2 — Обмер

Цель: полностью рабочий шаг 3 визарда — ввод всех замеров комнаты (стены, окна, двери, элементы, углы, фото), автосохранение, валидация. Seed-данные и тесты расчётов.

Пайплайн: `docs/agent-pipeline.md`

## S2-01. CRUD стен (WallsModule)

**Агент:** nestjs-expert

**Описание:**
Реализовать:
- `POST /api/rooms/:roomId/walls` — добавить стену.
- `PUT /api/walls/:id` — обновить (длина, материал, тип, кривизна в 3 точках).
- `DELETE /api/walls/:id` — удалить (каскадно удаляет окна, двери, adjacency).

DTO: `CreateWallDto`, `UpdateWallDto` с валидацией (длина > 0, материал из enum, кривизна в мм). `CreateWallDto` включает `cornerFrom` и `cornerTo` (id углов A и B соответственно). Label автоматически или вручную в формате "A-B".

**Тесты:**
- CRUD стен. Каскадное удаление. Валидация полей.

**Зависимости:** Sprint 1 завершён.

## S2-02. CRUD окон и дверей (OpeningsModule)

**Агент:** nestjs-expert

**Описание:**
Реализовать:
- Окна: `POST/PUT/DELETE /api/walls/:wallId/windows`. Поля: width, height, sillHeightFromScreed, revealWidthLeft/Right, isFrenchDoor.
- Двери: `POST/PUT/DELETE /api/walls/:wallId/doors`. Поля: width, heightFromScreed, revealLeft?, revealRight?. **Примечание:** поля connectsToRoomId нет — связь двери с комнатой определяется через WallAdjacency.

**Важное примечание:** простенки (пьедесталы между проёмами) теперь определяются через PLAIN-сегменты в WallSegment, а не через pierWidthLeft/pierWidthRight на проёмах. Эти поля удалены из DTO.

**Тесты:**
- CRUD окон и дверей. Валидация. Привязка к стене.

**Зависимости:** S2-01.

## S2-03. CRUD элементов комнаты (ElementsModule)

**Агент:** nestjs-expert

**Описание:**
Реализовать:
- `POST /api/rooms/:roomId/elements` — добавить элемент.
- `PUT /api/elements/:id` — обновить.
- `DELETE /api/elements/:id` — удалить.

**Тесты:**
- CRUD для каждого типа элемента. Валидация условных полей.

**Зависимости:** S2-01.

**Агент:** nestjs-expert

**Описание:**

## S2-04b. CRUD сегментов стен (SegmentsModule)

**Агент:** nestjs-expert

**Описание:**
Реализовать:
- `POST /api/walls/:wallId/segments` — добавить сегмент. Body: `CreateSegmentDto { segmentType, length, depth?, sortOrder, windowOpeningId?, doorOpeningId?, description? }`.
- `PUT /api/segments/:id` — обновить сегмент.
- `DELETE /api/segments/:id` — удалить сегмент.
- `POST /api/walls/:wallId/segments/validate` — валидировать сумму сегментов vs длина стены. Ответ: `{ wallLength, segmentsSum, difference, isValid }`. Допустимое расхождение ≤ 20мм.

Валидация при создании:
- segmentType = WINDOW → windowOpeningId обязателен.
- segmentType = DOOR → doorOpeningId обязателен.
- segmentType = PROTRUSION / NICHE → depth обязателен.
- sortOrder уникален в рамках стены.

**Тесты:**
- CRUD сегментов. Валидация суммы: корректная, расхождение 15мм (ОК), расхождение 25мм (ошибка). Проверка обязательных полей по типу сегмента.

**Зависимости:** S2-01, S2-02.

## S2-05. Загрузка фото (PhotosModule + FileStorageService)

**Агент:** nestjs-expert

**Описание:**
Реализовать:
- `POST /api/rooms/:roomId/photos` — загрузка фото комнаты (multipart). Query: `?type=OVERVIEW_BEFORE|OVERVIEW_AFTER|DETAIL`.
- `POST /api/elements/:elementId/photo` — фото элемента.
- `POST /api/windows/:windowId/photo` — фото окна.
- `POST /api/doors/:doorId/photo` — фото двери.
- `POST /api/angles/:angleId/photo` — фото угла.
- `DELETE /api/photos/:id` — удаление (из БД + с диска).
- `FileValidationPipe`: MIME (image/jpeg, image/png, image/webp), размер ≤ 10 МБ.
- **Расширение `FileStorageService`** (базовая версия создана в S1-05): добавить EXIF-очистку через `sharp`, генерацию thumbnail (400px по большей стороне), сохранение в `uploads/{userId}/{projectId}/{roomId}/`.
- Статика фото через Nginx (настройка location в nginx.conf).

**Тесты:**
- Загрузка валидного фото. Отклонение невалидного MIME. Отклонение > 10 МБ. Генерация thumbnail. Очистка EXIF. Удаление файла с диска.

**Зависимости:** S1-05, S2-01.

## S2-06. Seed-данные (тестовая квартира)

**Агент:** qa

**Описание:**
Создать `seed/test-apartment.json` на основе первого скриншота (двухкомнатная квартира с обмерного плана). Данные включают:
- Проект: название, тип.
- 4–5 комнат (большая комната, маленькая комната, коридор, санузел/ванная, кухня).
- Стены с реальными размерами из скриншота (в мм).
- Окна, двери с размерами.
- Радиатор (отопительный прибор, труба D30).
- Заранее посчитанные ожидаемые значения: площадь, периметр, объём каждой комнаты.

Создать скрипт `seed/seed.ts` — заливка данных в БД через Prisma.
Добавить команду `npm run seed`.

**Тесты:**
- Unit-тесты расчётов (RoomsCalcService): площадь, периметр, объём — сравнение с эталонными значениями из seed.
- Тест кривизны: среднее и отклонение.

**Зависимости:** S2-01, S2-02, S2-03.

## S2-07. Фронтенд: разметка углов, высота, габариты стен (шаг 3, подшаги 3.1–3.3)

**Агент:** react-expert

**Описание:**
Реализовать фронтенд для подшагов 3.1–3.3 обмера помещения:

**3.1. Разметка углов:**
- UI-компонент для расстановки углов: A (левый нижний), далее B, C, D... по часовой стрелке.
- Шаблон формы комнаты (из шага 2) — пользователь подтверждает или добавляет углы.
- Для прямоугольной — 4 угла, для L-shape — 6, для U-shape — 8.
- Визуальная подсказка: схема с буквами углов.

**3.2. Высота потолка:**
- 2 поля ввода: у окна + у противоположной стены.
- Предупреждение при разнице > 10мм.

**3.3. Габариты стен:**
- Для прямоугольных: ввод A-B и B-C (остальные = противоположным, автозаполнение).
- Для сложных форм: ввод всех стен (A-B, B-C, C-D, D-E, E-F, F-A).
- Для каждой стены: материал (выпадающий список), тип (внешняя/внутренняя/смежная).
- React Hook Form + Zod.

Zustand store: `useRoomMeasureStore` (текущая комната, текущий подшаг, данные формы).

**Тесты:**
- Рендеринг подшагов 3.1–3.3. Валидация форм. Навигация. Автозаполнение при прямоугольной форме.

**Зависимости:** Sprint 1 (S1-07, S1-11), S2-01, S2-05.

## S2-07b. Фронтенд: обход периметра и проёмы (шаг 3, подшаги 3.4–3.5)

**Агент:** react-expert

**Описание:**
Реализовать фронтенд для подшагов 3.4–3.5 обмера помещения:

**3.4. Обход периметра:**
- UI-компонент: интерактивная схема комнаты с текущей стеной, подсвеченной.
- Пользователь добавляет сегменты один за другим: тип (стена/окно/дверь/выступ/ниша/перегородка), длина, глубина (для выступов/ниш).
- Визуальная обратная связь: полоска-прогресс заполнения стены (сумма сегментов / длина стены).
- Автовалидация: если сумма сегментов расходится с длиной стены > 20мм — предупреждение с предложением перемерить.
- Нумерация сегментов (1, 2, 3...) как на обмерном плане.

**3.5. Проёмы и откосы:**
- Для каждого окна (из WINDOW-сегментов): высота, высота подоконника от стяжки, откосы (глубина лево/право), фото.
- Для каждой двери (из DOOR-сегментов): высота от стяжки, откосы, фото.
- Пометка для французских/балконных дверей.
- Ширина окна/двери уже заполнена из длины сегмента (только чтение).

React Hook Form + Zod.

**Тесты:**
- Рендеринг подшагов 3.4–3.5. Валидация. Добавление/удаление сегментов. Проверка валидации суммы.

**Зависимости:** S2-07, S2-02, S2-04b.

## S2-07c. Фронтенд: развёртка стен, элементы и финал (шаг 3, подшаги 3.6–3.7)

**Агент:** react-expert

**Описание:**
Реализовать фронтенд для подшагов 3.6–3.7 обмера помещения:

**3.6. Развёртка стен:**
- UI-компонент `WallElevationView`: «полотно» стены (прямоугольник: длина × высота потолка).
- Автоматически отображает проёмы из сегментов (позиция X из суммы предыдущих сегментов, высота из WindowOpening/DoorOpening).
- Пользователь добавляет вертикальные привязки: верх окна до потолка, высота двери от пола.
- Пользователь размещает элементы на развёртке: радиатор, щиток, вент-шахта, стояк — drag & drop или ввод координат (positionX от начала стены, offsetFromFloor от пола).
- Для угловых элементов: пометка «угловой, привязан к углу X» (cornerLabel).
- Вертикальные размеры выступов/ниш из обхода периметра.
- Кривизна стены: 3 поля ввода (низ, середина, верх). Автоматический расчёт среднего и отклонения.
- Последовательный обход: A-B → B-C → C-D → D-A.

**3.7. Фото + чеклист:**
- Обязательное фото общего вида комнаты.
- Чек-лист: все стены замерены? Все проёмы? Все элементы на развёртках? Кривизна проверена?
- Кнопка «Готово с комнатой» (активна если все обязательные поля заполнены).

React Hook Form + Zod.

**Тесты:**
- Рендеринг подшагов 3.6–3.7. Валидация развёртки. Drag & drop элементов. Чек-лист правильно показывает пропуски. Кнопка заблокирована при неполных данных.

**Зависимости:** S2-07b, S2-03.
