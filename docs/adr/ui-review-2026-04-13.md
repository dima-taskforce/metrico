**Дата:** 2026-04-13

> Ревью проведено без live браузера — на основе исходного кода. Визуальные баги требуют проверки в браузере.

## Мобильная версия

### UI-M-01 — Wizard: нет индикатора прогресса на мобильных (P1)

**Страницы:** `WallStep`, `CornerLabelStep`, `SketchStep`  
Многошаговый wizard не имеет видимого прогресс-бара или breadcrumbs. На мобильном пользователь не понимает, сколько шагов осталось.  
**Исправление:** Добавить `ProgressBar` компонент с номером шага (Шаг 2 из 5).

### UI-M-02 — LoginPage: кнопки OAuth могут быть слишком узкими на маленьких экранах (P2)

**Файл:** `client/src/pages/LoginPage.tsx:86`  
Три кнопки (Google, Яндекс, ВКонтакте) в `flex-col` с `w-full` — OK для мобильного. Но контейнер `max-w-sm` (384px) без `px-4` на мобильном → кнопки вплотную к краю экрана.  
**Исправление:** Добавить `px-4` к `min-h-screen` контейнеру.

### UI-M-03 — ProjectsPage: таблица проектов не адаптирована для мобильных (P1)

Требует проверки — если используется `table`, на мобильных (<640px) горизонтальный скролл или обрезание.  
**Исправление:** Использовать card-layout на мобильных или горизонтальный скролл с `overflow-x-auto`.

### UI-M-04 — TouchTarget: кнопки в wizard могут быть меньше 44px (P2)

Минимальный рекомендуемый touch target — 44×44px (Apple HIG / WCAG 2.5.5). Кнопки с `px-4 py-2` могут быть ~36px высотой на мобильных.  
**Исправление:** Увеличить padding до `py-3` для кнопок в wizard на мобильных.

## Десктопная версия

### UI-D-01 — Wizard: нет способа вернуться на предыдущий шаг (P1)

**Файл:** `client/src/pages/wizard/`  
В wizard шаги только двигаются вперёд. Нет кнопки «Назад» для исправления ошибок. Критично для длинного wizard (измерение комнат).  
**Исправление:** Добавить кнопку «← Назад» в каждый шаг wizard, управляющую `step` в `WizardPage`.

### UI-D-02 — ProjectsPage: нет пустого состояния (empty state) (P2)

Новые пользователи без проектов видят пустую страницу. Нет иллюстрации и CTA-кнопки «Создать первый проект».  
**Исправление:** Добавить `EmptyState` компонент с иконкой и текстом.

### UI-D-03 — Нет обработки ошибок загрузки данных на страницах проектов (P1)

Если API возвращает 5xx, пользователь видит пустую страницу без объяснений.  
**Исправление:** Добавить `ErrorBoundary` или inline error state с кнопкой «Повторить».

### UI-D-04 — Нет Loading skeleton для проектов (P2)

При загрузке данных нет skeleton-плейсхолдеров — страница «прыгает» при появлении контента.  
**Исправление:** Добавить `SkeletonCard` компоненты для списка проектов.

### UI-D-05 — LoginPage: нет ссылки «Забыл пароль» (P2)

**Файл:** `client/src/pages/LoginPage.tsx`  
Форма входа не содержит ссылку на `/forgot-password`. Пользователи OAuth-аккаунтов и локальных пользователей не могут восстановить доступ без этой ссылки.  
**Исправление:** Добавить ссылку `Забыли пароль?` под полем пароля.

### UI-D-06 — RegisterPage: нет подтверждения пароля (P2)

Форма регистрации запрашивает пароль один раз. Пользователи часто делают опечатки.  
**Исправление:** Добавить поле `Повторите пароль` с cross-field validation через zod `.refine()`.

### UI-D-07 — Нет глобального Toast/Notification для успешных операций (P2)

После сохранения измерений, создания проекта — нет визуального подтверждения успеха. Пользователь не уверен, сохранилось ли.  
**Исправление:** Добавить `useToast` hook + `Toaster` компонент (можно использовать `react-hot-toast`).

### UI-D-08 — Sketch/FloorPlan: нет кнопки «Отмена» (Undo) (P1)

В редакторе плана/схемы нет undo. Случайное удаление или перемещение элемента — необратимо.  
**Исправление:** Реализовать `useUndoStack` hook с историей действий (минимум 10 шагов).

## Доступность (Accessibility)

### UI-A-01 — SVG-иконки в кнопках OAuth без `aria-label` (P2)

**Файл:** `client/src/pages/LoginPage.tsx`  
Кнопки содержат SVG с `aria-hidden="true"` + текст. Порядок корректный. Но у самих `<a>` нет `role="button"` и `aria-label` для screen readers.  
**Исправление:** Добавить `aria-label="Войти через Google"` к OAuth ссылкам.

### UI-A-02 — Нет `focus-visible` стилей на custom компонентах (P3)

Tailwind `focus:ring-2` добавлен на нескольких элементах, но не на всех интерактивных. Навигация с клавиатуры может быть неудобной.
