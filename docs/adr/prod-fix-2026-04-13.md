# ADR: Исправление prod-конфигурации — 2026-04-13

**Статус:** Готово к применению (ожидает одобрения)  
**Сервер:** venus (`ssh venus`), `/opt/metrico`

---

## Проблема

Prod-сервер `metrico360.ru` запущен с **`docker-compose.yml`** (dev-конфиг) вместо **`docker-compose.prod.yml`**.

### Диагностика

```bash
# На сервере:
$ docker compose ps
# → app запущен, но с NODE_ENV=development

# Curl тест:
$ curl -c /tmp/c -b /tmp/c https://metrico360.ru/api/auth/login \
    -H "Content-Type: application/json" \
    -d '{"email":"test@test.com","password":"testtest1"}'
# → Set-Cookie: access_token=...; Path=/; HttpOnly; SameSite=Lax
#   ОТСУТСТВУЕТ: Secure flag
```

### Последствия

| Параметр | Dev (текущий) | Prod (правильный) |
|----------|--------------|-------------------|
| NODE_ENV | `development` | `production` |
| Cookie Secure | ❌ нет | ✅ есть |
| CORS_ORIGINS | `http://localhost:5173` | `https://metrico360.ru` |
| DATABASE_URL | `file:./prisma/metrico.db` | `file:/app/data/metrico.db` |
| OAuth callbacks | localhost | metrico360.ru |

---

## Решение

### Шаг 1: Убедиться, что `.env.production` актуален

Файл `/opt/metrico/.env.production` должен содержать:
```env
JWT_SECRET=<сгенерировать: openssl rand -hex 64>
JWT_REFRESH_SECRET=<сгенерировать: openssl rand -hex 64>
GOOGLE_CLIENT_ID=<из Google Cloud Console>
GOOGLE_CLIENT_SECRET=<из Google Cloud Console>
YANDEX_CLIENT_ID=<из oauth.yandex.ru>
YANDEX_CLIENT_SECRET=<из oauth.yandex.ru>
VK_CLIENT_ID=<из vk.com/editapp>
VK_CLIENT_SECRET=<из vk.com/editapp>
```

### Шаг 2: Применить prod-конфиг

```bash
cd /opt/metrico
git pull origin main
docker compose down
docker compose -f docker-compose.prod.yml --env-file .env.production up -d --build app
```

### Шаг 3: Применить миграцию БД

```bash
docker compose -f docker-compose.prod.yml --env-file .env.production \
  exec app npx prisma migrate deploy
```

### Шаг 4: Проверка

```bash
# Health check
curl https://metrico360.ru/api/health

# Проверить cookie флаги
curl -v -c /tmp/c -b /tmp/c https://metrico360.ru/api/auth/login \
    -H "Content-Type: application/json" \
    -d '{"email":"test@test.com","password":"testtest1"}' 2>&1 | grep -i "set-cookie"
# Ожидаем: Secure присутствует
```

## Обновление `deploy.yml`

Текущий GitHub Actions `deploy.yml` деплоит через `docker compose up` без указания файла. Нужно обновить:

```yaml
# .github/workflows/deploy.yml
# Найти строку с docker compose up и заменить на:
docker compose -f docker-compose.prod.yml --env-file .env.production pull && \
docker compose -f docker-compose.prod.yml --env-file .env.production up -d --no-build app
```

> После этого каждый деплой будет автоматически использовать prod-конфиг.

---

## Риски

- **Downtime:** `docker compose down` → ~30 сек пока поднимается новый контейнер
- **Data:** БД в volume `data` — данные сохранятся
- **Rollback:** Если что-то пойдёт не так, вернуться к dev-конфигу: `docker compose up -d`
